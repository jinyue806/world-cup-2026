# WC26 世界杯投注账本 — 交接文档

## 项目概述

零依赖对话式世界杯投注 CLI Skill。朋友间赌球，说一句话就记上。

## 设计理念

**Agent 优先，CLI 降级**：
- 用户只说自然语言，Agent 自动执行
- CLI 命令是实现细节，用户不需要知道
- 所有操作通过对话完成

## 快速开始

用户第一次提到世界杯/投注时，Agent 自动初始化：
1. 检查 `data/` 目录是否存在
2. 不存在则执行 `ensureDataDir()` + `ensureStandingsFresh()`
3. 告诉用户："世界杯账本已初始化，104 场比赛已加载。可以开始下注了。"

## 对话场景

| 用户说 | Agent 做 |
|:-------|:---------|
| 韩国赢捷克 100 | 精确匹配 → 查赔率 → 下注 |
| 巴西赢 200 | 模糊匹配 → 多场则列出候选 → 下注 |
| 下注 | 显示所有可下注比赛 → 用户选择 |
| 韩国 2-1 捷克 | 更新比分 → 自动结算 |
| 我赢了多少 | 显示盈亏统计 |
| 看看 A 组 | 显示小组积分榜 |
| 今天的热搜 | 获取微博热搜 |
| 删掉韩国那注 | 删除指定注单 |
| 我要重置 | 重置所有数据（需确认） |
| 存入 500 | 充值本金 |
| 导入下注 | 批量导入下注记录 |
| 淘汰赛预测 | 显示淘汰赛对阵 + 预测 |

## Agent 执行规则

1. **赔率自动获取**：下注时从投注平台 API 获取实时赔率
2. **比赛自动匹配**：根据用户说的队伍名自动查找比赛 ID
3. **比分自动结算**：更新比分后自动结算该比赛的所有注单
4. **积分榜自动刷新**：超过 30 分钟自动从 API 刷新
5. **结算通知**：比分更新后自动检查并生成结算通知

## 边界情况处理

### 多场比赛匹配
- 用户说"韩国赢"，但韩国有多场比赛 → 列出所有韩国的比赛，让用户选择
- 用户说"巴西赢"，巴西只有一场未结束比赛 → 自动匹配该比赛

### 淘汰赛特殊规则
- 加时赛胜者 → 按 90 分钟比分结算，加时赛不计入
- 点球大战胜者 → 按 90 分钟比分结算，点球不计入

### 数据安全
- 重置数据 → 必须用户确认，显示影响范围
- 删除注单 → 确认具体注单，避免误删
- 批量操作 → 显示操作预览，确认后执行
